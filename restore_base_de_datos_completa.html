RESTORE BASE DE DATOS COMPLETA

Para hacer el ejercicio de restore de la base de datos completa, vamos a hacer un backup completo, incluidos los archivelogs.

Ahora, para simular un daño en la base de datos, vamos a borrar todos los archivos de la carpeta DATAFILES, de ORADATA de la base de datos. (Mucho cuidado con borrar el Control File y los Redo Log Files. Solamente vamos a borrar los archivos de datos, los .DBF).

Al tratar de hacer un SHUTDOWN IMMEDIATE, Oracle detecta la ausencia de dichos archivos, sacando un error.

Al verificar el archivo de TRACE de la base de datos, vemos que el error es debido a la falta de estos archivos de datos. 

Es importante, antes de hacer un recovery de la base de datos, consultar el archivo de TRACE para mirar la causa del error.

Entonces hagamos un SHUTDOWN ABORT. Y luego hacemos un STARTUP MOUNT. Esto lo podemos hacer sin problemas, porque aun existe el Control File.
